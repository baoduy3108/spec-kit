# Real-World Metaphors in UI

Using familiar physical-world patterns to make interfaces intuitive and reduce cognitive load.

## Key Patterns

**Cards** work best for discrete, comparable content items. They should feel "graspable" with visual boundaries and shadow. One primary action per card keeps interaction clear.

**Carousels/horizontal scrolling** suit browsing scenarios where users explore naturally ordered content. However, they hide items — important content shouldn't live exclusively within them.

**Drawers** slide from edges to reveal secondary details without replacing the main view. They work well for mobile navigation and contextual editing.

**Accordions, tabs, and tooltips** follow similar logic: accordions collapse hierarchical content; tabs organize peer-level sections; tooltips provide brief contextual hints without cluttering layouts.

## Core Principles

1. Match metaphors to actual interactions — a non-clickable card creates false affordance
2. Avoid mixing multiple metaphors in one space
3. Adapt metaphors to platform (swipe gestures suit mobile; desktop interaction differs)
4. Use visual depth (shadow, overlay) to reinforce metaphorical hierarchy

Use them deliberately, not decoratively. A well-chosen pattern feels natural immediately; misalignment between visual metaphor and actual behavior confuses users.
