# Documentation Skill (Diátaxis Framework)

This skill helps create and restructure technical documentation using the **Diátaxis framework**, which organizes content into four distinct types based on user needs.

## The Four Documentation Types

**Tutorials** (Learning-oriented)
- Use when: user is new and learning by doing
- Structure: Goal → Prerequisites → Numbered steps → Verifiable results
- Key principle: "Minimise explanation; maximise doing"

**How-to Guides** (Problem-oriented)
- Use when: user needs to solve a specific, understood problem
- Structure: Goal → Assumptions → Steps → Expected result
- Key principle: assume baseline knowledge; allow for variation

**Reference** (Information-oriented)
- Use when: user needs to look up technical facts
- Structure: consistent format (name → type → default → description → example)
- Key principle: facts must be findable in under 30 seconds

**Explanations** (Understanding-oriented)
- Use when: user wants conceptual background or rationale
- Structure: Context → Core concept → Trade-offs → Higher perspective
- Key principle: focus on the *why*, not step-by-step *how*

## How to Use This Skill

1. Ask clarifying questions first about context, audience, and goals
2. Identify which type fits the user's need using the decision checklist
3. Apply type-specific patterns for structure and tone
4. Keep types separate — don't mix tutorial steps with reference tables
5. Cross-link between types to create an integrated system
6. Validate using the type-specific checks before delivery
