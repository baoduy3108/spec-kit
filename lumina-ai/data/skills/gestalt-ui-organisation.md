# Gestalt UI Organisation

Organize UIs so visual structure communicates which elements belong together, without requiring users to read documentation.

## Key Principles

**Proximity:** elements that are close together are perceived as a group. Related controls should cluster tightly; unrelated groups need larger spacing. Whitespace is preferred over separator lines.

**Similarity:** visually consistent elements (colour, shape, size) signal functional relationships. Destructive actions should look visually distinct from constructive ones.

**Common Region:** cards, panels, or background fills group logically related content. Borderless regions need generous internal padding and clear separation from neighbours.

**Connectedness:** lines and connectors show relationships only when they genuinely exist — used sparingly, e.g. navigation trees or node editors.

**Figure/Ground:** interactive controls need sufficient contrast; disabled or contextual information should recede visually.

**Continuity:** aligning related controls along consistent axes reinforces groupings through spatial logic.

## Quick Review Checklist

- Can users identify related controls without reading labels?
- Is proximity the primary grouping signal?
- Are unrelated groups separated by meaningful whitespace?
- Do unnecessary divider lines get replaced with spacing?
- Do destructive actions look visually distinct?

Common anti-patterns: scattered related controls, overused dividers, inconsistent icon styles — all solvable through deliberate spacing and visual consistency.
