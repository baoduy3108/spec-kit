# TypeScript Magician

This skill specializes in **advanced TypeScript type challenges**, focusing on eliminating `any` types and crafting type-safe solutions.

## Core Purpose

Use when addressing:
- TypeScript compiler errors and type inference problems
- Removing `any` types with proper alternatives
- Complex generics, conditional types, and type manipulation
- Strict typing requirements and type safety issues

## Key Workflow

1. **Capture errors** — identify the exact compiler error/message
2. **Identify root causes** (unsound inference, missing constraints, implicit `any`)
3. **Craft precise solutions** using advanced TS features
4. **Validate fixes** conceptually (would compile cleanly)

## Main Capabilities

- Advanced generics and conditional type logic
- Template literal and mapped type transformations
- Utility types (`Partial`, `Record`, `ReturnType`, `Awaited`, etc.)
- Brand/opaque types for nominal typing
- Complex inference patterns and type narrowing
- Module augmentation and declaration merging
- Function overloads and type guards

## Approach

Explain the type theory behind the fix, offer multiple solution approaches when relevant, show before/after comparisons, and note trade-offs (e.g. readability vs strictness) of each approach.
